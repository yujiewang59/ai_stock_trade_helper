# 股票综合分析Agent Skill

## 1. Skill 基础信息

|项目|详情|
|---|---|
|Skill 唯一标识|`synthesis-analysis`|
|适用场景|股票投资全维度决策辅助，整合基本面价值分析、技术面择时分析、市场情感分析三大维度，结合行业特性动态调整维度权重，输出综合投资决策|
|核心能力|1. 行业差异化维度权重动态分配；2. 多维度决策结果的标准化融合；3. 子分析置信度加权的综合决策生成；4. 多维度信号一致性的置信度量化；5. 缺失维度的自动降级处理|
|输入要求|股票基础信息、基本面 / 技术面 / 情感分析三大维度的独立分析结果|
|输出格式|结构化文本 / JSON 格式，固定输出：综合决策、决策置信度、综合决策原因|
|外部依赖|无强依赖，可直接对接`base-analysis`、`technical-analysis`、`sentiment-analysis`三大子 Skill 的输出结果|

## 2. 输入参数定义

### 2.1 必选参数

|参数名|类型|字段说明|取值要求|
|---|---|---|---|

1. `stock_name`：股票名称

3. `industry`：所属行业，支持取值：消费 / 科技 / 医药 / 周期 / 金融 / 房地产 / 公用事业，未匹配行业默认使用通用模型|
4. `risk_level`：投资者风险偏好，用于调整综合决策阈值，支持取值：low（低风险）/ medium（中风险，默认）/ high（高风险）|

1. `decision`：基本面决策，可选：buy/hold/sell

3. `reason`：基本面分析原因|
|`技术面分析结果：`|Dict|技术面分析的独立结果|必须包含 3 个核心字段：
1. `decision`：技术面决策，可选：buy/hold/sell

3. `reason`：技术面分析原因|
|`市场情绪分析结果：`|Dict|市场情感分析的独立结果|必须包含 3 个核心字段：
1. `decision`：情感面决策，可选：buy/hold/sell
2. `confidence`：情感面决策置信度，0~1 之间
3. `reason`：情感面分析原因|

## 3. 核心处理逻辑

### 3.1 行业差异化维度权重分配规则

根据不同行业的投资驱动特性，动态调整**基本面、技术面、情感分析**三大维度的权重占比（权重总和固定为 100%），适配各行业的核心决策逻辑，具体配置如下：

|行业类型|基本面权重|技术面权重|情感分析权重|行业适配核心逻辑|
|---|---|---|---|---|
|**消费行业**
(白酒 / 食品 / 家电 / 零售)|50%|35%|15%|消费股长期价值由基本面决定，技术面辅助择时，市场情绪影响较小，优先锚定基本面价值|
|**科技行业**
(电子 / 计算机 / 半导体 / 高端制造)|30%|35%|35%|科技股行情受技术择时与市场情绪驱动显著，基本面为长期支撑，短期优先关注技术信号与市场情绪|
|**医药行业**
(创新药 / 医疗器械 / 医疗服务)|40%|30%|30%|医药股受政策情绪与事件驱动影响大，基本面为长期安全垫，技术面辅助择时，兼顾情绪信号|
|**周期行业**
(有色 / 钢铁 / 化工 / 煤炭 / 资源)|40%|40%|20%|周期股行情由基本面周期位置与技术面拐点共同驱动，情绪仅为辅助，均衡配置基本面与技术面权重|
|**金融行业**
(银行 / 证券 / 保险 / 多元金融)|55%|30%|15%|金融股核心看基本面资产质量，技术面辅助择时，市场情绪影响有限，优先锚定基本面安全边际|
|**房地产行业**
(地产开发 / 物业 / 建材)|45%|30%|25%|地产股受政策情绪驱动显著，基本面合规性为安全垫，技术面辅助判断企稳信号，兼顾情绪与基本面|
|**公用事业**
(电力 / 水务 / 燃气 / 环保)|60%|30%|10%|公用事业股盈利稳定、高分红，长期价值完全由基本面决定，情绪影响极小，优先锚定基本面稳定性|
|**通用行业**
(未匹配行业默认)|40%|35%|25%|无明确行业特征的标的，采用均衡权重配置，兼顾价值、择时与情绪的均衡性|
### 3.2 子分析结果标准化处理

将三大维度的独立决策结果，统一转换为**0~1 区间的标准化得分**，用于后续加权计算：

|子分析决策|标准化得分|说明|
|---|---|---|
|buy（买入）|1.0|该维度下，多头信号明确，支持买入|
|hold（持有）|0.5|该维度下，多空信号平衡，支持持有|
|sell（卖出）|0.0|该维度下，空头信号明确，支持卖出|
### 3.3 综合得分计算与决策生成

#### 3.3.1 加权综合得分公式

考虑子分析的置信度，自动弱化低可靠度子分析的影响，公式如下：

```Plain Text

分子 = (基本面得分 * 行业基本面权重 * 基本面置信度) + 
       (技术面得分 * 行业技术面权重 * 技术面置信度) + 
       (情感得分 * 行业情感权重 * 情感置信度)
分母 = (行业基本面权重 * 基本面置信度) + 
       (行业技术面权重 * 技术面置信度) + 
       (行业情感权重 * 情感置信度)
综合得分 = 分子 / 分母
```

- 综合得分固定范围：0~1，得分越高代表综合多头信号越强

#### 3.3.2 决策映射规则

根据用户风险偏好动态调整阈值，与子 Skill 的阈值规则对齐，保证决策一致性：

|风险偏好|buy（买入）阈值|hold（持有）区间|sell（卖出）阈值|
|---|---|---|---|
|低风险（low）|综合得分 ≥ 0.75|0.45 ≤ 得分 < 0.75|综合得分 < 0.45|
|中风险（medium，默认）|综合得分 ≥ 0.70|0.40 ≤ 得分 < 0.70|综合得分 < 0.40|
|高风险（high）|综合得分 ≥ 0.65|0.35 ≤ 得分 < 0.65|综合得分 < 0.35|
- 买入决策：多维度信号共振，基本面价值支撑、技术面择时信号、市场情绪三者共振，上涨动能充足

- 持有决策：多维度信号存在分歧，无明确的多空共振信号，处于震荡整理阶段，无显著操作机会

- 卖出决策：多维度空头信号共振，基本面走弱、技术面破位、市场情绪低迷，下跌风险显著

### 3.4 综合决策置信度计算规则

置信度范围固定为**0~1**，数值越高代表综合决策的可靠性越强，核心基于 3 个维度加权计算，公式如下：

```Plain Text

置信度 = (子分析一致性 * 0.5) + (子分析平均置信度 * 0.3) + (行业匹配度 * 0.2)
```

1. **子分析一致性（权重 50%）**：三大维度决策的一致性水平，信号越一致，决策可靠性越高

    - 三大维度决策完全一致，一致性得分 = 1.0

    - 两大维度决策一致，一致性得分 = 0.7

    - 三大维度决策完全分歧，一致性得分 = 0.3

2. **子分析平均置信度（权重 30%）**：三大子分析自身置信度的算术平均值，子分析越可靠，综合决策越可靠

3. **行业匹配度（权重 20%）**：行业的适配程度，主流行业的权重配置更精准，通用行业存在偏差

    - 7 大主流行业，匹配度得分 = 1.0

    - 未匹配通用行业，匹配度得分 = 0.9

4. **缺失维度扣减**：每缺失 1 个子分析结果，置信度自动下调 0.15，缺失超过 2 个则终止分析

## 4. 边界约束与合规声明

1. **适用范围边界**：本 Skill 仅针对 A 股市场公开可获取的股票数据进行综合决策辅助，整合三大维度的分析结果，不涉及内幕信息、高频交易决策。

2. **合规强制声明**：本 Skill 输出的所有结果仅为投资辅助参考，**不构成任何投资建议或操作指导**，股票市场有风险，投资需谨慎。

3. **数据有效性边界**：仅处理合理范围内的输入数据，子分析决策 / 置信度超出合理范围的异常值将直接拦截，不进行分析。

4. **维度适配边界**：仅对 7 大主流行业提供精准差异化权重，其他未匹配行业使用通用模型，决策置信度自动下调 0.1。

5. **禁止使用场景**：不得用于杠杆交易、配资交易、高频交易的决策依据，不得用于误导性投资宣传。

## 5. 错误处理与降级策略

### 5.1 输入校验错误处理

|错误场景|错误码|处理方式|
|---|---|---|
|缺失股票所属行业信息|`MISSING_INDUSTRY`|终止分析，返回提示：「请提供股票所属行业，以匹配差异化综合分析权重」|
|缺失所有子分析结果|`MISSING_ALL_RESULTS`|终止分析，返回提示：「请提供至少 1 项子分析结果，方可完成综合分析」|
|子分析决策 / 置信度值超出合理范围|`INVALID_SUB_RESULT`|终止分析，返回提示：「输入的子分析结果存在异常，请检查数据正确性后重新输入」|
### 5.2 运行时错误降级策略

|异常场景|降级策略|
|---|---|
|缺失 1 个子分析结果（开启自动降级）|自动将剩余 2 个子分析的权重重新归一化，置信度下调 0.15，在决策原因中明确提示：「缺失 XX 维度分析结果，已自动调整权重完成分析，决策置信度已下调」|
|缺失 2 个子分析结果（开启自动降级）|自动使用剩余 1 个子分析的结果作为综合决策，置信度下调 0.3，在决策原因中明确提示：「缺失 2 项维度分析结果，仅基于剩余维度完成分析，决策置信度已大幅下调」|
|未匹配行业使用通用模型|自动使用通用均衡权重，置信度下调 0.1，在决策原因中明确提示：「所属行业不在适配范围内，已使用通用综合分析模型，决策置信度已下调」|

## 6. 标准输出格式规范

输出必须为结构化格式，固定包含核心字段，支持文本与 pydantic 两种标准化输出：

### 6.1 标准文本输出格式

```Plain Text

# 股票综合分析结果
1. 股票标的：【股票名称(股票代码)】
2. 所属行业：【行业类型】
4. 综合投资决策：【buy/hold/sell】
5. 决策置信度：【0~1之间的数值，保留2位小数】
6. 决策原因：
   - 【分点说明三大子分析的核心结论，结合行业权重逻辑，说明综合决策依据】
   - 【提示潜在风险与信号分歧点】
```

### 6.2 标准 JSON 输出格式

```json

{
    "stock_name": "股票名称",
    "stock_code": "股票代码",
    "industry": "所属行业",
    "decision": "buy/hold/sell",
    "confidence": 0.XX,
    "decision_reason": {
        "core_basis": ["子分析结论1", "子分析结论2", "综合决策依据"],
        "risk_tips": ["风险提示1", "风险提示2"]
    }
}
```
